# 实习上手清单

入职后可以按这个顺序熟悉工作。

## 第 1 天：先别急着改脚本

先搞清楚：

- 项目根目录在哪里
- 工具环境怎么 source
- flow 入口命令是什么
- 日志放在哪里
- 成功和失败怎么判断

## 看一个 flow 时问自己

- 输入是什么
- 输出是什么
- 中间日志在哪里
- 哪一步最容易失败
- 失败后看哪个 report
- 这个步骤由 Shell、Tcl 还是 Python 控制

## 遇到失败怎么办

建议顺序：

1. 看终端最后 20 行。
2. 看 `logs/flow.log`。
3. 找第一个出现 `ERROR/FAIL` 的地方。
4. 看对应步骤的 report。
5. 确认输入文件和配置有没有变。
6. 再考虑是不是工具或 license 问题。

## 向导师提问的好方式

不推荐：

```text
这个失败了，怎么办？
```

推荐：

```text
我跑 testchip flow，在 route 步骤失败。
日志位置是 xxx/logs/flow.log。
第一个错误是 xxx。
我检查了输入 RTL 和 SDC 都存在。
我怀疑是配置里的 xxx 参数问题，请问我应该继续查哪个 report？
```
